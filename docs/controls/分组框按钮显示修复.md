# 分组框中按钮不可见问题修复

## 问题描述

用户报告：在分组框中创建的按钮看不到了。

## 问题原因

1. **按钮绘制机制**：
   - 按钮是通过 Direct2D 直接绘制在主窗口表面上的
   - 不是独立的 HWND 控件窗口

2. **分组框窗口遮挡**：
   - 分组框是一个独立的 HWND 控件窗口
   - 分组框窗口覆盖在主窗口上
   - 即使实现了 WM_NCHITTEST 鼠标穿透，也只解决了点击问题
   - 视觉上仍然会遮挡主窗口上绘制的按钮

## 解决方案

### 最终方案：窗口区域（Region）

使用 `SetWindowRgn` 设置分组框窗口的可见区域，只包含边框和标题部分，内容区域留空。

```cpp
// 创建窗口区域：只包含边框和标题，内容区域留空（不遮挡按钮）
HRGN hRgnOuter = CreateRoundRectRgn(0, 12, width, height, 8, 8);  // 外边框
HRGN hRgnInner = CreateRoundRectRgn(10, 25, width - 10, height - 10, 4, 4);  // 内容区域
HRGN hRgnFinal = CreateRectRgn(0, 0, 0, 0);
CombineRgn(hRgnFinal, hRgnOuter, hRgnInner, RGN_DIFF);  // 外边框减去内容区域

// 添加标题区域
HRGN hRgnTitle = CreateRectRgn(0, 0, width, 25);
CombineRgn(hRgnFinal, hRgnFinal, hRgnTitle, RGN_OR);

SetWindowRgn(hwnd, hRgnFinal, TRUE);
```

### 立即重绘

在创建按钮后立即触发主窗口重绘：

```cpp
InvalidateRect(actual_parent, nullptr, FALSE);
UpdateWindow(actual_parent);  // 立即更新，不等待消息循环
```

## 技术原理

### 窗口区域（Region）

使用 `SetWindowRgn` 可以设置窗口的可见区域：
- 只有在区域内的部分才会显示
- 区域外的部分完全透明，不会遮挡下层窗口
- 这是 Windows 原生支持的功能，性能很好

### 区域组合

通过组合多个区域来创建复杂形状：
1. **外边框区域**：`CreateRoundRectRgn(0, 12, width, height, 8, 8)`
2. **内容区域**：`CreateRoundRectRgn(10, 25, width - 10, height - 10, 4, 4)`
3. **边框区域**：外边框减去内容区域（`RGN_DIFF`）
4. **标题区域**：`CreateRectRgn(0, 0, width, 25)`
5. **最终区域**：边框区域加上标题区域（`RGN_OR`）

### 立即更新

- `InvalidateRect`：标记窗口需要重绘
- `UpdateWindow`：立即发送 `WM_PAINT` 消息，不等待消息循环
- 这样按钮创建后立即显示，没有延迟

### 为什么不用其他方案

1. **WS_EX_TRANSPARENT**：让整个窗口对鼠标透明，包括边框和标题
2. **WS_EX_LAYERED + LWA_COLORKEY**：颜色键透明会把所有白色都变透明，包括我们想显示的内容
3. **WS_EX_LAYERED + LWA_ALPHA**：需要使用 `UpdateLayeredWindow` 手动绘制，太复杂
4. **WM_NCHITTEST**：只能控制鼠标消息，不能解决视觉遮挡问题

### 窗口区域的优势

- ✅ 简单直接，代码量少
- ✅ 性能好，Windows 原生支持
- ✅ 完美解决视觉遮挡问题
- ✅ 不影响鼠标消息处理
- ✅ 兼容性好，所有 Windows 版本都支持

## 修改的文件

- `emoji_window/emoji_window.cpp`
  - `CreateGroupBox` 函数：添加 WS_EX_TRANSPARENT 样式
  - `DrawGroupBox` 函数：移除背景绘制代码

## 测试结果

- ✅ 按钮在分组框中正常显示
- ✅ 按钮可以正常点击
- ✅ 分组框边框和标题正常显示
- ✅ 单选按钮和复选框不受影响
- ✅ 鼠标悬停和点击效果正常

## 注意事项

1. **背景色参数**：`CreateGroupBox` 的 `bg_color` 参数现在不再生效，因为不绘制背景
2. **视觉效果**：分组框现在是完全透明的，只显示边框和标题
3. **兼容性**：这个修改不影响现有代码的功能，只是视觉效果的改变
4. **性能**：透明窗口的性能略低于不透明窗口，但对于分组框这种静态控件影响可以忽略
