# 分组框控件改进说明

## 改进概述

从本版本开始，分组框（GroupBox）支持作为父窗口直接创建子控件，无需手动添加子控件到分组框。

## 问题分析

### 旧版本的问题

1. **使用繁琐**：需要两步操作
   - 先在窗口坐标系中创建控件
   - 再手动调用 `添加子控件到分组框`

2. **坐标系统混乱**：
   - 子控件坐标是相对于主窗口的
   - 不是相对于分组框的
   - 计算复杂，容易出错

3. **代码冗余**：
   ```易语言
   ' 旧方式 - 需要手动计算窗口绝对坐标
   单选按钮1 = 创建单选按钮_辅助(窗口句柄, 40, 60, 100, 30, "男", 1, 真, #黑色, #白色)
   添加子控件到分组框(分组框1, 单选按钮1)  ' 手动添加
   ```

## 新版本改进

### 核心改进

1. **分组框可作为父窗口**：直接将分组框句柄传递给子控件创建函数
2. **自动坐标转换**：系统自动将分组框相对坐标转换为窗口绝对坐标
3. **自动管理子控件**：无需手动调用 `添加子控件到分组框`

### 实现原理

在子控件创建函数（如 `CreateRadioButton`、`CreateCheckBox`）中：

```cpp
// 检查父窗口是否为分组框
auto groupbox_it = g_groupboxes.find(hParent);
if (groupbox_it != g_groupboxes.end()) {
    // 父窗口是分组框，需要转换坐标
    GroupBoxState* gb_state = groupbox_it->second;
    actual_parent = gb_state->parent;  // 使用分组框的父窗口
    
    // 转换坐标：分组框相对坐标 -> 窗口绝对坐标
    actual_x = gb_state->x + x + 10;  // 左边距10px
    actual_y = gb_state->y + y + 25;  // 标题高度约25px
    
    // 创建后自动添加到分组框
    AddChildToGroup(hParent, hwnd);
}
```

### 坐标说明

- **分组框内容区域**：从标题下方约 25px 处开始
- **左边距**：默认 10px
- **坐标原点**：分组框内容区域的左上角

```
┌─────────────────────────┐
│  分组框标题 (高度25px)   │
├─────────────────────────┤
│ (10px边距)              │
│    ↓ (0,0) 坐标原点     │
│    子控件从这里开始      │
│                         │
└─────────────────────────┘
```

## 透明背景修复（v1.1）

### 问题描述

在之前的版本中，按钮在分组框中创建后不可见或无法点击。原因是：

1. **按钮绘制方式**：按钮是通过 D2D 直接绘制在主窗口上的（不是 HWND 控件）
2. **分组框窗口遮挡**：分组框是一个独立的 HWND 控件窗口，会覆盖在主窗口上
3. **视觉遮挡**：分组框窗口遮挡了主窗口上绘制的按钮

### 解决方案

1. **移除背景填充**：只绘制边框和标题，不绘制背景填充
   ```cpp
   // 不绘制背景，保持完全透明，这样主窗口上的按钮就能显示出来
   ```

2. **透明背景清除**：在 WM_PAINT 中使用完全透明的背景
   ```cpp
   render_target->Clear(D2D1::ColorF(D2D1::ColorF::White, 0.0f));
   ```

3. **精确的鼠标命中测试**：在 `WM_NCHITTEST` 中精确控制哪些区域可以接收鼠标消息
   ```cpp
   case WM_NCHITTEST: {
       // 将屏幕坐标转换为客户区坐标
       POINT pt = { GET_X_LPARAM(lparam), GET_Y_LPARAM(lparam) };
       ScreenToClient(hwnd, &pt);
       
       // 计算边框和标题区域
       int title_height = 25;
       int border_width = 10;
       
       // 如果点击在边框或标题区域，返回 HTCLIENT
       bool in_border_or_title = 
           pt.y < title_height ||  // 标题区域
           pt.x < border_width ||  // 左边框
           pt.x > client_rect.right - border_width ||  // 右边框
           pt.y > client_rect.bottom - border_width;  // 下边框
       
       if (in_border_or_title) {
           return HTCLIENT;  // 边框和标题可以接收消息
       }
       
       // 内容区域返回 HTTRANSPARENT，让主窗口上的按钮能被点击
       return HTTRANSPARENT;
   }
   ```

4. **立即重绘**：创建分组框后立即触发重绘，确保显示
   ```cpp
   InvalidateRect(hwnd, nullptr, TRUE);
   UpdateWindow(hwnd);
   ```

### 技术原理

- **不使用 WS_EX_TRANSPARENT**：这个样式会让整个窗口对鼠标完全透明，包括边框和标题
- **精确控制透明区域**：通过 `WM_NCHITTEST` 消息，只让内容区域透明，边框和标题仍然可以接收消息
- **HTTRANSPARENT 的作用**：让鼠标消息穿透到下层窗口（主窗口），这样主窗口上的按钮就能接收点击

### 效果

- ✅ 分组框立即显示，不需要鼠标移动触发
- ✅ 按钮在分组框中正常显示
- ✅ 按钮可以正常点击
- ✅ 分组框边框和标题正常显示
- ✅ 不影响其他子控件（单选按钮、复选框等）

## 注意事项

1. **坐标原点**：子控件坐标 (0,0) 对应分组框内容区域左上角
2. **边距处理**：系统自动添加 10px 左边距和 25px 顶部边距
3. **自动管理**：使用新方式时，无需也不应该手动调用 `添加子控件到分组框`
4. **按钮回调**：即使按钮在分组框中创建，回调参数中的父窗口句柄仍然是主窗口句柄（不是分组框句柄）
5. **透明背景**：分组框背景现在是完全透明的，只显示边框和标题。如果需要背景色，应该在主窗口上绘制
