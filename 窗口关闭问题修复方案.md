# 自绘窗口关闭问题修复方案

## 问题描述

用户点击按钮1创建自绘窗口后，关闭易语言窗口1时，自绘窗口不会自动关闭。

## 根本原因分析

1. **自绘窗口是独立的顶层窗口**，有自己的消息循环（`运行消息循环()`）
2. **易语言窗口和自绘窗口是独立的**，各有各的消息循环
3. **关键问题**：主窗口的 `WindowProc` 没有处理 `WM_CLOSE` 消息！

## 已尝试的方案

### ❌ 方案 1：直接调用销毁窗口
```易语言
.子程序 _窗口1_将被销毁
.如果真 (窗口句柄 ≠ 0)
    销毁窗口 (窗口句柄)  ' ❌ 失败：消息循环还在运行
.如果真结束
```

### ❌ 方案 2：使用 PostMessage 发送 WM_CLOSE
```易语言
.子程序 _窗口1_将被销毁
.如果真 (窗口句柄 ≠ 0)
    PostMessage (窗口句柄, WM_CLOSE, 0, 0)  ' ❌ 失败：WindowProc 不处理 WM_CLOSE
.如果真结束
```

### ❌ 方案 3：使用 SendMessage 发送 WM_CLOSE
```易语言
.子程序 _窗口1_将被销毁
.如果真 (窗口句柄 ≠ 0)
    SendMessage (窗口句柄, WM_CLOSE, 0, 0)  ' ❌ 失败：WindowProc 不处理 WM_CLOSE
.如果真结束
```

## ✅ 正确的解决方案

### 步骤 1：在 C++ 端添加 WM_CLOSE 消息处理

在 `emoji_window/emoji_window.cpp` 的 `WindowProc` 函数中，在 `WM_DESTROY` 之前添加：

```cpp
case WM_CLOSE:
    // 处理窗口关闭消息
    // 调用 DestroyWindow 会触发 WM_DESTROY，进而调用 PostQuitMessage
    DestroyWindow(hwnd);
    return 0;

case WM_DESTROY:
    PostQuitMessage(0);
    return 0;
```

### 步骤 2：在易语言端使用 SendMessage 发送 WM_CLOSE

```易语言
.子程序 _窗口1_将被销毁

' ⚠️ 重要：先销毁 TabControl，再关闭自绘窗口
.如果真 (TabControl句柄 ≠ 0)
    销毁TabControl (TabControl句柄)
    TabControl句柄 ＝ 0
.如果真结束

' ⚠️ 重要：使用 SendMessage 同步发送 WM_CLOSE 消息关闭自绘窗口
' SendMessage 是同步的，会等待窗口处理完消息
.如果真 (窗口句柄 ≠ 0)
    SendMessage (窗口句柄, WM_CLOSE, 0, 0)
    窗口句柄 ＝ 0
.如果真结束
```

### 步骤 3：添加必要的 DLL 命令和常量

在 `易语言代码/DLL命令.e` 中添加：

```易语言
.DLL命令 SendMessage, 整数型, "user32.dll", "SendMessageA"
    .参数 窗口句柄, 整数型
    .参数 消息, 整数型
    .参数 wParam, 整数型
    .参数 lParam, 整数型
```

在 `易语言代码/常量表.e` 中添加：

```易语言
.常量 WM_CLOSE, "16", , 关闭窗口消息（0x0010）
```

## 工作原理

1. **易语言窗口关闭时**：触发 `_窗口1_将被销毁` 事件
2. **发送 WM_CLOSE 消息**：使用 `SendMessage` 同步发送到自绘窗口
3. **WindowProc 处理 WM_CLOSE**：调用 `DestroyWindow(hwnd)`
4. **触发 WM_DESTROY**：`DestroyWindow` 会自动发送 `WM_DESTROY` 消息
5. **退出消息循环**：`PostQuitMessage(0)` 使 `运行消息循环()` 返回
6. **窗口正常关闭**：自绘窗口完全关闭

## 关键点

1. **必须在 WindowProc 中处理 WM_CLOSE**：这是标准的 Windows 窗口关闭流程
2. **使用 SendMessage 而不是 PostMessage**：确保消息被同步处理
3. **先销毁 TabControl**：避免子控件残留
4. **WM_CLOSE → DestroyWindow → WM_DESTROY → PostQuitMessage**：这是正确的关闭顺序
