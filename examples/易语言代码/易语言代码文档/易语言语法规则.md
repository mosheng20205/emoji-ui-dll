# 易语言语法规则

## 0. DLL 编译位数规则

- 给易语言示例使用的 `emoji_window.dll`，默认按 `32 位 / Win32` 编译。
- 如果是给易语言工程直接加载 DLL，优先输出 `Release|Win32` 版本，不要默认只编译 `x64`。
- 交付前需要确认以下两个位置的 DLL 都是 32 位版本：
  - `emoji_window.dll`
  - `examples/易语言代码/4.0/emoji_window.dll`
- 如果同时存在 32 位和 64 位构建产物，以易语言实际加载的 `Win32` DLL 为准。

本文档已同步 `.kiro/steering/testing-guidelines.md`、`.kiro/steering/易语言 Emoji 表情使用注意事项.md`、`.kiro/steering/易语言规则.md` 的要求，用于统一本项目中的易语言示例代码写法、UTF-8 处理方式和测试规范。

## 1. 条件判断语法

易语言的条件判断有两种形式，不能混用。

### 1.1 单分支判断

没有 `.否则` 时，使用 `.如果真` / `.如果真结束`：

```e
.如果真 (条件)
    ...
.如果真结束
```

### 1.2 双分支判断

只要出现 `.否则`，必须使用 `.如果` / `.否则` / `.如果结束`：

```e
.如果 (条件)
    ...
.否则
    ...
.如果结束
```

### 1.3 严禁混用

下面这种写法是错误的：

```e
.如果真 (条件)
    ...
.否则
    ...
.如果结束
```

结论：

- 没有 `.否则`：用 `.如果真` / `.如果真结束`
- 有 `.否则`：用 `.如果` / `.否则` / `.如果结束`

## 2. 变量地址与指针传递规则

### 2.1 不要使用 `取字节集指针`

本项目中传递字节集缓冲区地址时，正确命令是 `取变量数据地址`，不要使用 `取字节集指针`。

错误示例：

```e
标签_演示 = 创建标签 (主窗口句柄, 50, 30, 700, 60, 取字节集指针 (utf8字节集), 取字节集长度 (utf8字节集), 到ARGB (255, 255, 255, 255), 到ARGB (255, 70, 130, 180), 取文本指针 ("微软雅黑"), 取文本字节数 ("微软雅黑"), 16, 真, 假, 假, 1, 假)
```

正确示例：

```e
标签_演示 ＝ 创建标签 (主窗口句柄, 50, 30, 700, 60, 取变量数据地址 (utf8字节集), 取字节集长度 (utf8字节集), 到ARGB (255, 255, 255, 255), 到ARGB (255, 70, 130, 180), 取文本指针 (“微软雅黑”), 取文本字节数 (“微软雅黑”), 16, 真, 假, 假, 1, 假)
```

### 2.2 不要使用 `取字节集数据地址`

正确命令仍然是 `取变量数据地址`。

错误示例：

```e
获取标签字体 (标签_演示, 取字节集数据地址 (字体名缓冲区), 字体名长度, 字体大小, 粗体, 斜体, 下划线)
```

正确示例：

```e
获取标签字体 (标签_演示, 取变量数据地址 (字体名缓冲区), 字体名长度, 字体大小, 粗体, 斜体, 下划线)
```

## 3. 按钮回调与按钮 ID 规则

### 3.1 按钮点击回调参数必须完整

参考：`docs/controls/button.md`

错误示例：

```e
.子程序 按钮点击回调, , , stdcall
.参数 按钮ID, 整数型
```

正确示例：

```e
.子程序 按钮点击回调, , , stdcall
.参数 按钮ID, 整数型
.参数 _父窗口句柄, 整数型
```

### 3.2 按钮 ID 必须保存到程序集变量

`创建Emoji按钮_字节集` 返回的按钮 ID 是动态分配的，不保证从 1 开始连续递增。不能在回调中用硬编码数字判断按钮。

错误示例：

```e
' 创建时丢弃返回值
创建Emoji按钮_字节集 (主窗口句柄, ...)

' 回调中用硬编码数字判断
.判断开始 (按钮ID ＝ 1)
    ' 处理逻辑
.判断 (按钮ID ＝ 2)
    ' 处理逻辑
.判断结束
```

正确示例：

```e
' 程序集变量保存按钮ID
.程序集变量 按钮_获取文本, 整数型
.程序集变量 按钮_获取字体, 整数型

' 创建时保存返回值
按钮_获取文本 ＝ 创建Emoji按钮_字节集 (主窗口句柄, ...)
按钮_获取字体 ＝ 创建Emoji按钮_字节集 (主窗口句柄, ...)

' 回调中用程序集变量判断
.判断开始 (按钮ID ＝ 按钮_获取文本)
    ' 处理逻辑
.判断 (按钮ID ＝ 按钮_获取字体)
    ' 处理逻辑
.判断结束
```

### 3.3 位运算命令

易语言没有 `位左移` 命令，正确命令是 `左移`。

## 4. 符号书写规则

易语言中赋值、运算、比较等符号必须使用全角字符，半角符号会导致语法错误。

| 用途 | 错误（半角） | 正确（全角） |
|------|--------------|--------------|
| 赋值/等于 | `=` | `＝` |
| 加法/拼接 | `+` | `＋` |
| 大于 | `>` | `＞` |
| 小于 | `<` | `＜` |
| 大于等于 | `>=` | `≥` |
| 小于等于 | `<=` | `≤` |
| 不等于 | `!=` | `≠` |

## 5. UTF-8 字节集与 DLL 文本处理规则

`设置标签文本`、`设置按钮文本` 等接口接收的是 UTF-8 字节集指针。凡是 DLL 返回的文本缓冲区，都应按 UTF-8 字节集处理，不能当作 ANSI 文本直接转换。

### 5.1 禁止用 `到文本()` 或 `编码_Utf8到Ansi()` 处理 DLL 返回缓冲区

错误示例：

```e
' ❌ 到文本() 按 ANSI 解释 UTF-8 字节，中文/emoji 会乱码
文本内容 = 到文本 (文本缓冲区)
结果信息 = "文本内容：" + 文本内容
设置标签文本 (标签_结果, 取文本指针 (结果信息), 取文本字节数 (结果信息))

' ❌ 编码_Utf8到Ansi() 同样会导致乱码
文本内容 = 到文本 (编码_Utf8到Ansi (文本缓冲区))
```

正确示例：

```e
' ✅ 结果声明为字节集，全程用 UTF-8 字节集拼接
.局部变量 结果字节集, 字节集

' DLL 返回的缓冲区直接参与拼接，不做任何转换
结果字节集 = 编码_Ansi到Utf8 ("文本内容：") ＋ 文本缓冲区 ＋ { 10 } ＋ 编码_Ansi到Utf8 ("字节长度：") ＋ 编码_Ansi到Utf8 (到文本 (文本长度))

' 用取变量数据地址传给设置标签文本
设置标签文本 (标签_结果, 取变量数据地址 (结果字节集), 取字节集长度 (结果字节集))
```

### 5.2 换行符使用字节 `{ 10 }`

在字节集拼接中，换行用 `{ 10 }` 直接插入，不要使用 `#换行符`，因为 `#换行符` 是文本型常量。

## 6. Emoji 表情使用注意事项

### 6.1 问题来源

易语言 IDE 使用 ANSI 编码，不支持 Unicode 特殊符号。代码中直接写 emoji（如 `"🎨"`）在 IDE 中会变成问号 `?`，编译后传给 DLL 的也仍然是问号，最终导致 emoji 丢失。

### 6.2 正确做法

将 emoji 以 UTF-8 原始字节集的形式硬编码，中文部分用 `编码_Ansi到Utf8()` 转换，两者使用 `＋` 拼接。

写法对比：

```e
' ❌ 错误 - emoji 经过 ANSI 编码会变成问号
utf8字节集 = 到UTF8字节集 ("🎨 自定义主题")

' ✅ 正确 - emoji 用字节集，中文用编码转换，拼接
utf8字节集 = { 240, 159, 142, 168, 32 } ＋ 编码_Ansi到Utf8 ("自定义主题")
'              🎨 emoji UTF-8字节    空格
```

### 6.3 拼接规则

| 部分 | 处理方式 | 示例 |
|------|----------|------|
| Emoji | 直接写 UTF-8 字节数组 `{ ... }` | `{ 240, 159, 142, 168 }` |
| 空格 | 字节 `32` | 放在 emoji 字节集末尾 |
| 中文/ASCII 文本 | `编码_Ansi到Utf8("文本")` | `编码_Ansi到Utf8 ("自定义主题")` |
| 拼接 | 用 `＋` 连接字节集 | `emoji字节集 ＋ 文本字节集` |

### 6.4 窗口标题也必须使用字节集版本

`创建Emoji窗口()` 的标题参数是 ANSI 文本，标题中如果包含 emoji，同样会丢失。应改用 `创建Emoji窗口_字节集()`。

错误示例：

```e
窗口句柄 = 创建Emoji窗口 ("主题切换示例 🎨", 600, 500)
```

正确示例：

```e
utf8字节集 = 编码_Ansi到Utf8 ("主题切换示例 ") ＋ { 240, 159, 142, 168 }
窗口句柄 = 创建Emoji窗口_字节集 (取变量数据地址 (utf8字节集), 取字节集长度 (utf8字节集), 600, 500)
```

## 7. 测试规范

### 7.1 测试语言优先级

本项目测试优先使用 Python 编写，原因如下：

1. Python 测试可跨平台运行
2. Python 代码简洁，便于维护
3. Python 无需编译，测试迭代速度快
4. Python 有丰富的测试与断言生态

### 7.2 测试文件位置

所有 Python 测试文件应放置在：`examples/Python/`

命名规范：

- 功能测试：`test_<功能名称>.py`
- 示例：`test_window_title.py`
- 示例：`test_button_properties.py`

### 7.3 测试文件结构

```python
"""
测试<功能名称>
简要描述测试内容
"""
import ctypes
from ctypes import wintypes
import sys
import os

# 添加当前目录到路径
sys.path.insert(0, os.path.dirname(__file__))

# 加载DLL
try:
    dll = ctypes.CDLL('./emoji_window.dll')
except OSError:
    print("错误: 无法加载emoji_window.dll")
    sys.exit(1)

# 定义函数原型
# ... 函数声明 ...

def test_功能名称():
    """测试主函数"""
    print("=" * 60)
    print("测试<功能名称>")
    print("=" * 60)

    # 测试用例
    # ...

    return True  # 或 False

if __name__ == "__main__":
    success = test_功能名称()
    sys.exit(0 if success else 1)
```

### 7.4 测试要求

每个测试至少应覆盖：

1. 正常情况
2. 边界情况
3. 异常情况
4. UTF-8 / 中文 / emoji 编码情况
5. 返回文本类接口的“两次调用模式”

另外还必须满足：

- 输出信息清晰，标明测试步骤与结果
- 成功返回 `0`，失败返回 `1`
- 正确清理资源，例如销毁窗口

### 7.5 何时使用 C++ 测试

仅在以下场景使用 C++ 测试：

1. 需要测试底层性能
2. 需要测试特定的 C++ API 行为
3. Python 不方便覆盖的场景

C++ 测试文件放在项目根目录，命名为 `test_<功能名称>.cpp`。

### 7.6 运行测试

```bash
cd examples/Python
python test_window_title.py
```

或：

```bash
cd examples/Python
run.bat
```

### 7.7 测试文档要求

每个测试文件头部应注明：

- 测试目的
- 测试功能点
- 预期结果
- 特殊说明（如果有）
