# 窗口大小改变功能重新实现说明

## 实现日期
2026年3月6日 14:32

## 背景

在之前的实现中，添加窗口大小改变回调功能时引入了 TabControl 布局更新问题：
1. 窗口大小改变后切换 Tab 时布局不正确
2. 出现递归调用导致窗口变空白
3. 切换 Tab 后内容变空白

因此回退到 commit 9302eac（修复编辑框和标签的闪烁问题），并重新实现窗口大小改变功能。

## 新的实现策略

### 核心思路
1. **添加窗口大小改变回调功能**（C++ 端）
2. **提供手动更新 TabControl 布局的函数**（不自动处理 WM_SIZE）
3. **在易语言中手动调用更新函数**（完全控制更新时机）

### 关键改进
- ✅ **避免在 C++ 中自动处理 TabControl 的 WM_SIZE**
- ✅ **提供独立的 `UpdateTabControlLayout` 函数供手动调用**
- ✅ **在易语言回调中手动更新布局**，避免递归调用

case WM_SIZE: {
    if (state) {
        UINT width = LOWORD(lparam);
        UINT height = HIWORD(lparam);
        state->render_target->Resize(D2D1::SizeU(width, height));
        
        // ⚠️ 调用窗口大小改变回调（如果设置了）
        if (g_window_resize_callback) {
            g_window_resize_callback(hwnd, width, height);
        }
    }
    return 0;
}
```

#### 实现导出函数

// 手动更新 TabControl 布局（窗口大小改变后调用）
void __stdcall UpdateTabControlLayout(HWND hTabControl) {
    auto it = g_tab_controls.find(hTabControl);
    if (it == g_tab_controls.end()) return;

    TabControlState* state = it->second;
    UpdateTabLayout(state);
}

// 设置窗口大小改变回调
void __stdcall SetWindowResizeCallback(WindowResizeCallback callback) {
    g_window_resize_callback = callback;
}
```

## 工作原理

### 窗口大小改变流程
1. **用户拖动窗口边缘** → 触发主窗口 `WM_SIZE` 消息
2. **WindowProc 处理 WM_SIZE** → 调用 `g_window_resize_callback`
3. **易语言回调函数被调用** → `窗口大小改变回调(窗口句柄, 新宽度, 新高度)`
4. **易语言中手动调用** → `更新TabControl布局(TabControl句柄)`
5. **C++ 更新布局** → `UpdateTabControlLayout` → `UpdateTabLayout`
6. **所有 Tab 页布局更新完成**

### Tab 切换流程
1. **用户点击 Tab** → 触发 `TCN_SELCHANGE` 通知
2. **TabControlParentSubclassProc 接收通知** → 自动调用 `UpdateTabLayout`
3. **布局更新完成** → 显示当前 Tab，隐藏其他 Tab
4. **触发易语言回调**（如果设置了）

## 关键优势

### 1. 避免递归调用
- ❌ **旧方案**：在 `TabControlParentSubclassProc` 中处理 `WM_SIZE` → 调用 `UpdateTabLayout` → 触发新的 `WM_SIZE` → 递归
- ✅ **新方案**：只在易语言回调中手动调用 `UpdateTabControlLayout`，完全控制更新时机

### 2. 完全控制更新时机
- 易语言代码可以决定何时更新 TabControl 布局
- 可以在更新前后添加额外的逻辑
- 避免了 C++ 端的自动处理带来的副作用

### 3. 保持原有功能
- Tab 切换时仍然自动更新布局（通过 `TCN_SELCHANGE` 通知）
- 不影响其他功能的正常工作
- 保持了代码的简洁性
